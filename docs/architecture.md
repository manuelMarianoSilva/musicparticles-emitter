## Notes

### Dead Code

- **`TouchEventSerializer.fromJson()`** — Fully implemented but never called anywhere in the
  codebase. `BroadcastServer` only calls `toJson()`. The deserialization path was likely written
  in anticipation of a receiving side that was never built. Safe to delete or mark `@Deprecated`.

### Structural Concerns

- **No error boundary on the GL thread** — `ParticleRenderer.onDrawFrame()` calls `ps.update()`
  inside a `synchronized` block but swallows no exceptions. A crash on the GL thread will silently
  kill rendering with no user feedback.

- **Tight coupling via callbacks in `MainActivity`** — The `onBurstThreshold` lambda captures
  `surfaceView` by reference before `surfaceView` is fully initialised (`lateinit`). If the burst
  timer fires during `onCreate` before `setContentView` returns there is a potential
  `UninitializedPropertyAccessException`.

- **`ParticleRenderer` renders at most 12 000 particles** (`MAX_PART = 12_000`) while
  `ParticleSystem` holds up to 20 000. Particles beyond 12 000 are silently dropped on each
  frame.

- **`TouchEventType` enum values use the prefix `TOUCH_`** (`TOUCH_DOWN`, `TOUCH_MOVE`, …) but
  the network JSON serialises them as their `.name` string — meaning the receiving side on LAN
  must know the prefixed names. Consider stripping the prefix for a cleaner wire format.

- **Boilerplate test stubs** — `ExampleUnitTest.kt` and `ExampleInstrumentedTest.kt` contain
  only the auto-generated "1 + 1 == 2" placeholder tests. No real unit or integration tests exist
  for any production class.
